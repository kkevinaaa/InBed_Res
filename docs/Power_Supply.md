# 供电方案说明

## 1. 当前供电路线

当前供电部分分两条路线推进。第一条路线使用现成 TP4056 充电模块做整机功耗测试，重点测量模块原始静态电流，以及拆除指示灯和限流电阻后的空载、带载电流。该路线用于快速判断现成模块是否适合第一版原型。

第二条路线使用已经完成的 `Cube Power Manager` 电源管理板。该板使用 USB-C 作为 5V 输入，TP4054 负责单节锂电池充电，AP2112K-3.3 负责输出 3.3V，给 MCU、屏幕和传感器供电。相比“电池升压到 5V 再降压到 3.3V”的方案，这一路径更直接，损耗更低，也更接近最终整机结构。

## 2. 方案一：TP4056 模块测试

TP4056 模块作为快速验证方案，优点是便宜、易买、接线简单，适合先测整机平均电流、峰值电流和大致续航。缺点是模块通常带有充电/满电指示灯和额外电阻，会产生持续静态耗电，不一定适合低功耗设备长期使用。

模块到货后先测原始状态下的静态电流，再拆除 LED 和相关限流电阻后重新测量。若拆除指示灯后静态电流足够低，可以作为早期原型供电方案；若静态电流仍然偏大，则只作为测试工具，不进入最终设计。

## 3. 方案二：Cube Power Manager

`Cube Power Manager` 是当前自制供电板方案。它集成 USB-C 输入、单节锂电池充电、电池供电路径、3.3V 稳压输出、电池电压检测和测试点，目标是验证最终设备所需的基础电源链路。

USB-C 只用于取 5V 电源，不做数据通信。CC1 和 CC2 使用下拉电阻声明设备端，从而让普通 USB-C 电源正常输出 5V。USB 输入进入 VIN 后，一路供给充电芯片，另一路通过电源路径给系统供电。

TP4054 负责单节锂电池充电。PROG 电阻设定充电电流，CHRG 引脚可用于充电状态指示。电池端引出 VBAT，同时保留电池接口和测试点，方便测量充电状态、电池电压和充电电流。

系统电源路径使用 VIN、VBAT 和 VCC 三个核心节点。USB 插入时，系统优先由 VIN 供电；USB 拔出时，系统由电池供电。当前设计中使用肖特基二极管和 PMOS 构成基础电源切换路径，并预留通过二极管方式验证电源 OR 的可能性。

AP2112K-3.3 将系统电源稳压为 3.3V。3.3V 输出用于 MCU、OLED 和 SC7A20H 等低压器件。该方案避免先升压再降压，结构简单，但需要注意单节锂电电压接近 3.3V 时，LDO 会逐渐失去稳压余量。

BAT_ADC 用于检测电池电压。电池电压经过电阻分压后送入 MCU ADC，避免超过 MCU 引脚耐压。采样点并联小电容用于降低噪声和稳定 ADC 读数。

## 4. 测试点与后续验证

电源板应重点测试 VIN、VBAT、VCC、3V3、GND、BAT_ADC 和充电状态信号。测试顺序从 USB 输入开始，再检查充电芯片、电池端、电源路径、LDO 输出和 ADC 检测。

第一次上电时先不接 MCU 负载，单独确认 3.3V 输出正常。随后接入 MCU 和屏幕，观察启动瞬间是否掉压、LDO 是否发热、电池供电是否稳定、USB 插拔时系统是否异常复位。

后续若要优化低功耗，优先检查三处：充电模块或指示灯带来的静态耗电，LDO 自身静态电流，以及电池电压检测分压电阻的持续耗电。
