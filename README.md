# 立方体番茄钟设计目标与功能概述

## 设计目标

制作一个小型立方体番茄钟设备。设备顶面放置一块小尺寸单色 OLED 屏幕，用于显示当前时间、番茄钟倒计时、电量和当前状态。与顶面相邻的四个侧面都可以作为“底面”与桌面接触，用户通过翻滚立方体 90 度来启动或切换计时状态。设备不设置实体按钮，主要通过姿态变化和摇晃动作进行交互。

设备应尽量体积小，适合 3D 打印外壳，便于拆卸。外壳预留 Type-C 接口，用于充电。整体目标是做成一个低功耗、无按钮、可翻滚交互的实体番茄钟。

## 核心功能

设备默认处于静息状态，屏幕显示当前时间和电量。在一段时间无操作后，屏幕自动熄灭，以节省电量。用户轻微晃动或移动设备时，屏幕重新点亮并回到时钟显示状态。

用户将立方体翻滚到指定侧面并稳定放置后，设备开始一次番茄钟计时，默认时长为 25 分钟。计时过程中，屏幕显示剩余时间和当前计时状态。计时结束后，设备激活蜂鸣器进行提醒，同时屏幕显示倒计时结束。提醒结束后，设备自动返回静息/时钟状态。

在计时过程中，番茄钟若朝向侧面翻滚，则暂停计时，屏幕提示番茄钟倒计时暂停后，设备恢复静息状态（时间显示）。

设备使用三轴加速度传感器 SC7A20H 检测当前姿态、翻滚动作和摇晃动作。姿态检测用于判断哪一个面朝下，快速摇晃用于进入特殊模式。快速摇晃进入蓝牙配对/设置模式这一点理论上可以实现：通过检测短时间内加速度模长的剧烈变化，或后续使用传感器中断功能识别明显运动事件。

## 时间与参数设置

设备不放置按钮，时间和参数设置通过手机 BLE 蓝牙连接完成。一般情况下，设备不主动开启蓝牙，以降低功耗。只有在用户快速摇晃设备后，设备才进入 BLE 配对/设置模式，并在屏幕上显示蓝牙设置状态。

手机连接后，可以设置当前时间、番茄钟默认时长、提醒震动方式、屏幕熄灭时间等参数。设置完成后，设备关闭 BLE，回到正常低功耗运行状态。

## 硬件方案

主控暂定使用 ESP32-C3。它可以读取加速度传感器、驱动 OLED 屏幕、控制震动芯片，并通过 BLE 与手机通信。开发阶段使用 PlatformIO + Arduino 框架。

屏幕选择 SSD1327 单色 OLED（128×64）小尺寸屏幕。OLED 适合黑底白字显示，没有 TFT 背光常亮的问题，更适合电池供电的小设备。

姿态检测使用 SC7A20H 三轴加速度传感器，通过 I2C 与 ESP32-C3 通信。第一阶段可以用轮询方式读取三轴数据，调试六面识别和摇晃检测；后续再使用 INT 引脚实现低功耗唤醒。

蜂鸣器使用12085封装 阻抗16R 使用mos管/三极管链接GPIO进行控制

电源使用单节锂电池方案，选择小型聚合物锂电池。充电部分使用 Type-C 单节锂电充电保护模块。电池输出再经过 5V 升压模块（或者可能使用LDO稳压模块降至3.3V），为 ESP32-C3、OLED、SC7A20H 供电。电量检测可以使用 MAX17048 电量计，或者第一版先用电阻分压接 ADC 做粗略检测。

## 低功耗策略

静息状态下关闭 OLED 显示，ESP32-C3 进入低功耗状态。SC7A20H 保持低功耗运动检测，用于检测晃动或翻滚事件。检测到明显运动后，唤醒主控，主控再读取完整三轴数据判断具体动作。

BLE 不常开，只在快速摇晃触发后短时间开启。若一段时间内没有手机连接或设置完成，设备自动关闭 BLE 并返回正常模式。

## 第一版实现范围

第一版优先实现以下功能：OLED 显示当前时间和倒计时；SC7A20H 读取三轴数据并识别六个方向；翻滚到指定面后开始 25 分钟计时；计时结束后蜂鸣提醒；快速摇晃进入 BLE 设置模式；无操作后自动熄屏。

第一版暂不追求极致小体积和最终外观，优先验证硬件接线、姿态交互、计时逻辑、BLE 设置和电源路径是否可行。外壳可以先做较大的 3D 打印版本，后续再优化内部布局和外观。
